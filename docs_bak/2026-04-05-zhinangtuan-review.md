## 三、需要明确的补充设计

### 3.1 讨论引擎 director.ts 决策矩阵

**开团条件检测**（出现分歧时制造冲突）：
- 两个角色对同一问题表态方向相反（赞成/反对）
- 触发方式：解析历史消息，检测矛盾关键词（如"但是"/"不对"/"我不同意"）

**邀请用户时机检测**：
- 出现观点分歧
- 角色质疑用户观点
- 讨论到达分叉点
- 出现意外信息或反转
- 讨论即将收敛

**收束条件检测**：
- 轮次达到上限（`maxTurnsPerRole * roles.length`）
- 用户发出结束信号（"总结"/"结束"/"结论"）
- 角色一致同意某观点（无分歧）

### 3.2 状态机转换条件

```
opening → developing
  条件：用户进入讨论，发出第一条消息
  动作：主持人介绍场景，各角色依次表态

developing → climax
  条件：检测到明显分歧 / 轮次过半
  动作：制造冲突升级，邀请用户参与

climax → closing
  条件：用户最终表态 / 轮次达到上限
  动作：主持人收束，总结各方观点
```

### 3.3 服务端接口扩展

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/templates` | GET | 返回模板列表（含版本号） |
| `/api/templates/:id` | GET | 返回完整模板 JSON，支持 ETag |
| `/api/models` | GET | 返回推荐免费模型列表 |
| `/api/health` | GET | 健康检查接口 |
| `/api/error` | - | 统一错误响应格式：`{ code, message, details }` |
