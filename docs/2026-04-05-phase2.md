# Phase 2: 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Phase 2 包含爽点机制升级、模板动态下发、产品化

**Architecture:** 基于 Phase 1 成果，扩展服务端能力和优化体验

**Tech Stack:** Next.js 14, Vercel Functions, CDN, Sentry

## Phase 2 Timeline

**Week 13-14: 爽点机制升级**
- 打脸机制升级：从关键词冲突检测升级为语义检测
- 使用 Embedding 或 LLM 裁判对比历史断言
- 爽点频率调优

**Week 15-16: 模板动态下发**
- 服务端模板接口实现（Vercel Functions）
- 客户端拉取、版本对比、ETag 缓存
- 离线可用
- 新模板 2: 创业公司董事会
- 新模板 3: 白雪公主智囊

**Week 17-18: 产品化**
- 多模型 Prompt 适配框架
- 免费模型体验优化
- 性能优化：首屏加载、流式渲染
- 移动端响应式适配
- 错误监控接入（Sentry）
- 用户数据埋点

## Tasks

### Task 1: 打脸机制语义升级 (Week 13)

**Files:**
- Modify: src/engine/events.ts
- Create: src/engine/semantic.ts
