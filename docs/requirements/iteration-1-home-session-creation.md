# 迭代 1：首页与创建讨论会话

## 迭代目标

实现首页主流程：用户填写议题，选择默认模板和模型策略，点击“开始讨论”后创建讨论会话，并进入讨论页。

本迭代不要求多 Agent 真实生成内容，只要求完成会话创建链路和首页交互闭环。

## 原型映射

| 原型区域 | 本迭代要求 |
|---|---|
| 首页 Hero | 议题输入、字数统计、开始讨论按钮 |
| 模板卡片 | 默认展示“三国军师团”，点击弹出模板选择 Sheet |
| 模型策略卡片 | 默认展示“智能躲避反馈”，点击弹出策略选择 Sheet |
| 快速开始 | 点击后填入议题 |
| 最近讨论 | 展示最近讨论，点击进入讨论页 |
| Toast / Sheet / Modal | 首页关键操作有反馈 |

## 前端需求

| 编号 | 需求 |
|---|---|
| FE-001 | 实现首页完整移动端布局，与原型结构一致 |
| FE-002 | 实现议题输入框，最大 100 字，实时字数统计 |
| FE-003 | 实现“开始讨论”按钮，空议题时提示错误 |
| FE-004 | 实现模板卡片，默认模板为“三国军师团” |
| FE-005 | 实现模板选择 Sheet，至少包含三国军师团、创业公司董事会、产品辩论桌三个选项 |
| FE-006 | 实现模型策略 Sheet，至少包含智能策略、质量优先、成本优先 |
| FE-007 | 实现快速开始议题按钮，点击后写入输入框 |
| FE-008 | 实现最近讨论列表，展示状态、模板、时间 |
| FE-009 | 点击最近讨论可进入对应讨论页 |
| FE-010 | 创建会话成功后跳转 `/discussion/[sessionId]` |

## 后端/API需求

| 编号 | 需求 |
|---|---|
| BE-001 | 实现 `POST /api/sessions` 创建会话 |
| BE-002 | 实现 `GET /api/sessions/recent` 查询最近会话 |
| BE-003 | 创建会话入参包含：topic、templateId、modelStrategyId |
| BE-004 | 创建会话返回：sessionId、topic、template、status、createdAt |
| BE-005 | 服务层实现 `SessionService.createSession` |
| BE-006 | Repository 层保存会话到 Mock Repository 或本地持久层 |
| BE-007 | 创建会话时初始化 DiscussionState，phase 初始为 `idle` |
| BE-008 | 支持默认模板数据读取 |
| BE-009 | 支持默认模型策略读取 |
| BE-010 | API 错误需包含明确错误码，例如 `TOPIC_REQUIRED`、`TEMPLATE_NOT_FOUND` |

## 验收标准

| 类型 | 标准 |
|---|---|
| 产品 | 用户打开首页能看到议题输入、模板、模型策略、快速开始、最近讨论 |
| 产品 | 用户点击快速开始后，输入框自动填入对应议题 |
| 产品 | 用户点击模板卡片能打开模板选择 Sheet |
| 产品 | 用户点击模型策略卡片能打开策略选择 Sheet |
| 产品 | 用户输入议题并点击开始讨论后进入讨论页 |
| 产品 | 空议题不能创建会话，并出现明确提示 |
| 技术 | `POST /api/sessions` 可成功创建会话 |
| 技术 | `GET /api/sessions/recent` 可返回最近会话 |
| 技术 | 创建会话时写入 session、topic、template、modelStrategy |
| 技术 | 首页不直接构造 sessionId，必须使用 API 返回值 |
| 技术 | 单元测试覆盖会话创建成功和失败场景 |
| 技术 | 页面加载、提交、错误状态均有展示 |

## 不包含范围

| 模板完整配置 | 迭代 8 实现 |

## 需求评审关注点

1. 首页是否严格按照原型实现，而不是重新设计。
2. 会话创建是否成为后续讨论链路的唯一入口。
3. 模板和模型策略是否先做“可选但简化”。
4. 最近讨论是否使用 API 数据，而不是组件内硬编码。
