# 迭代 4 PRD：讨论状态机与会话生命周期

## 1. 功能概述

本次迭代围绕“会话中心 + 讨论详情页 + 讨论状态机”补齐讨论生命周期闭环：用户可以在会话中心查看、搜索、筛选、恢复和归档已有讨论；点击会话后进入唯一的讨论详情路由 `/discussion/[sessionId]` 恢复上下文；讨论引擎具备 `idle → opening → developing → climax → closing` 的阶段推进能力，并能将讨论阶段与会话生命周期状态保持一致。

会话生命周期状态统一使用 `running / completed / archived`：已完成会话允许用户选择“继续讨论”后回到 `running`；讨论进入 `closing` 后，需等待主持人总结消息落库，再由用户人工确认完成后标记为 `completed`。

本迭代遵循最新导航基线：底部一级导航为 `首页 / 会话 / 模板 / 设置`，讨论页不作为一级 Tab，仅作为具体 session 的工作台存在。

## 2. Functional Requirements

### 会话中心

#### FR-001｜展示会话列表（P0）
- 功能点描述：用户进入“会话”页后，可以看到已有讨论会话列表。
- 输入：用户打开会话中心。
- 输出：列表展示会话标题、模板、状态、角色数、消息数或事件数、更新时间。
- 异常处理：当没有会话时，展示明确空状态，不显示空白页面或报错。

#### FR-002｜按状态筛选会话（P0）
- 功能点描述：用户可以按 `全部 / 进行中 / 已完成 / 已归档` 筛选会话。
- 输入：用户点击状态分段筛选项。
- 输出：列表仅展示匹配状态的会话。
- 异常处理：筛选结果为空时展示该筛选条件下的空状态。

#### FR-003｜搜索会话（P0）
- 功能点描述：用户可以通过搜索框查找会话。
- 输入：用户输入关键词。
- 输出：按会话标题、模板名称或角色信息匹配并展示结果。
- 异常处理：无匹配结果时展示“未找到相关会话”的空状态。

#### FR-004｜恢复已有会话（P0）
- 功能点描述：用户点击会话卡片后，进入对应讨论详情页并恢复该 session 的消息、角色和状态。
- 输入：用户点击某个会话卡片。
- 输出：页面跳转到 `/discussion/[sessionId]`，展示该会话的标题、模板、角色栏和消息流。
- 异常处理：如果 session 不存在或 sessionId 无效，展示错误提示，并引导用户返回首页或会话中心。

#### FR-005｜归档会话（P0）
- 功能点描述：用户可以将会话归档，归档后的会话从进行中/已完成列表中移出，并出现在已归档筛选下。
- 输入：用户对选中会话执行归档操作。
- 输出：会话状态更新为已归档，列表刷新后状态与筛选结果一致。
- 异常处理：重复归档同一会话时保持幂等，返回当前最终状态，不产生重复或错误状态。

#### FR-006｜恢复归档或完成会话（P1）
- 功能点描述：用户可以恢复可继续讨论的会话，使其回到进行中状态。
- 输入：用户对已归档或已完成会话执行恢复操作。
- 输出：会话状态更新为进行中，并可再次进入讨论详情页继续。
- 异常处理：无法恢复的会话应展示明确提示，不修改原状态。

#### FR-007｜新建会话入口（P1）
- 功能点描述：会话中心提供新建入口，用户点击后返回首页创建新讨论。
- 输入：用户点击“新建”入口。
- 输出：跳转到首页的议题输入与创建流程。
- 异常处理：跳转失败时提示用户重试。

### 首页最近讨论

#### FR-008｜首页最近讨论复用统一会话数据源（P0）
- 功能点描述：首页“最近讨论”展示与会话中心一致的数据，不使用独立 Mock 列表。
- 输入：用户打开首页。
- 输出：展示最近更新的会话，包含标题、模板、状态和更新时间。
- 异常处理：无最近会话时展示空状态或隐藏列表，不展示与会话中心不一致的数据。

#### FR-009｜从首页最近讨论进入讨论详情（P0）
- 功能点描述：用户点击首页最近讨论项时，进入对应 `/discussion/[sessionId]`。
- 输入：用户点击最近讨论卡片。
- 输出：恢复对应会话详情。
- 异常处理：如果会话已不存在，提示会话不可用并引导用户返回首页或会话中心。

### 讨论详情页

#### FR-010｜讨论页必须携带有效 sessionId（P0）
- 功能点描述：讨论页只能以 `/discussion/[sessionId]` 形式访问，不存在无 sessionId 的默认讨论页。
- 输入：用户访问讨论页路由。
- 输出：有效 sessionId 加载对应讨论；无效或缺失 sessionId 不进入默认讨论。
- 异常处理：无效路由展示错误状态和返回入口。

#### FR-011｜讨论详情页只承担当前会话工作台职责（P0）
- 功能点描述：讨论详情页展示当前会话消息流、角色栏、输入区和当前上下文，不提供跨会话列表、搜索、筛选或阶段评估入口。
- 输入：用户进入讨论详情页。
- 输出：用户看到当前会话的讨论内容，并可以继续输入消息。
- 异常处理：消息为空时展示可继续讨论的空/开场状态，不展示会话中心能力。

#### FR-012｜底部导航不展示讨论 Tab（P0）
- 功能点描述：底部一级导航只包含 `首页 / 会话 / 模板 / 设置`。
- 输入：用户查看任意主页面底部导航。
- 输出：不存在“讨论”一级 Tab。
- 异常处理：用户只能通过创建、最近讨论或会话卡片进入讨论详情页。

### 讨论状态机

#### FR-013｜支持讨论 phase 流转（P0）
- 功能点描述：系统支持 `idle → opening → developing → climax → closing` 的讨论阶段推进。
- 输入：会话创建、消息发送、讨论推进或收束动作。
- 输出：讨论 phase 按合法规则更新。
- 异常处理：非法跳转会被拒绝，并保留原 phase。

#### FR-014｜发送消息后推进讨论阶段（P0）
- 功能点描述：用户或角色发送消息后，系统根据当前 phase 和基础规则推进讨论阶段。
- 输入：当前会话状态、当前 phase、消息历史。
- 输出：phase 可能进入下一合法阶段，并更新状态历史。
- 异常处理：推进失败时保留当前 phase，并返回可诊断错误信息或降级为当前阶段继续。

#### FR-015｜会话状态与讨论 phase 保持关联但不混用（P0）
- 功能点描述：`status` 表示会话生命周期，`phase` 表示讨论阶段，两者应在服务层关联但保持语义独立。
- 输入：phase 进入 closing 且主持人总结消息已落库、用户归档、用户恢复、用户人工确认完成。
- 输出：主持人总结消息落库后，用户可人工确认完成并将会话标记为 `completed`；归档不改变历史 phase；恢复后 status 可回到 `running`。
- 异常处理：状态更新重复提交时保持幂等，返回当前最终状态。

#### FR-016｜记录状态历史（P1）
- 功能点描述：系统记录 phase 或 status 变化历史，便于调试和后续 Director 复盘。
- 输入：每次合法状态变化。
- 输出：记录 from、to、触发原因和时间。
- 异常处理：历史记录失败不应破坏当前会话主状态，但应保留可观察错误。

### API / 服务能力

#### FR-017｜查询会话列表（P0）
- 功能点描述：系统提供会话列表查询能力，支持 status、keyword、limit 参数。
- 输入：status、keyword、limit。
- 输出：按更新时间排序返回匹配会话列表。
- 异常处理：参数非法时返回校验错误；无结果时返回空列表。

#### FR-018｜更新会话状态（P0）
- 功能点描述：系统提供会话状态更新能力，支持 archive、complete、resume。
- 输入：sessionId 和目标动作。
- 输出：返回更新后的会话状态。
- 异常处理：session 不存在返回 NOT_FOUND；动作非法返回 VALIDATION_ERROR；重复动作幂等返回当前状态。

#### FR-019｜查询讨论状态（P0）
- 功能点描述：系统提供根据 sessionId 获取讨论状态的能力。
- 输入：sessionId。
- 输出：返回当前 phase、状态历史和与会话状态相关的必要信息。
- 异常处理：session 不存在返回 NOT_FOUND。

## 3. Non-Functional Requirements

### NFR-001｜一致性（P0）
首页最近讨论、会话中心和讨论详情页必须使用同一会话数据源，避免出现同一会话在不同页面状态、标题或更新时间不一致。

### NFR-002｜可测试性（P0）
讨论状态机必须独立于 UI，所有合法和非法 phase 转换都应可通过单元测试验证；会话查询、筛选、归档和恢复应可通过 API 或服务层测试验证。

### NFR-003｜可用性（P0）
无效 sessionId、空列表、搜索无结果、归档后列表为空等场景必须有用户可理解的提示，不允许白屏或静默失败。

### NFR-004｜移动端一致性（P1）
会话中心和讨论详情页的主要布局应符合移动端原型基线：会话页包含分段筛选、搜索框、会话卡片和底部操作区；讨论详情页包含顶部标题、角色栏、消息流和输入区。

### NFR-005｜性能（P1）
会话列表默认按更新时间分页或限制数量加载，避免一次渲染过多历史会话导致移动端卡顿。

## 4. 验收标准

### 对应 FR-001
- Given 用户进入会话中心，When 存在会话数据，Then 页面展示会话标题、模板、状态、角色数、消息数/事件数和更新时间。
- Given 当前没有任何会话，When 用户进入会话中心，Then 页面展示空状态。

### 对应 FR-002
- Given 会话列表中存在进行中、已完成、已归档会话，When 用户切换筛选项，Then 列表只展示对应状态的会话。
- Given 某筛选项无结果，When 用户切换到该筛选项，Then 展示该状态下的空状态。

### 对应 FR-003
- Given 会话列表中存在匹配标题、模板或角色的会话，When 用户输入关键词，Then 列表展示匹配结果。
- Given 无匹配结果，When 用户输入关键词，Then 展示无结果提示。

### 对应 FR-004 / FR-010
- Given 用户点击有效会话卡片，When 跳转完成，Then URL 为 `/discussion/[sessionId]`，并展示对应会话内容。
- Given 用户访问无效 sessionId，When 加载失败，Then 页面展示错误提示和返回入口。
- Given 用户尝试访问无 sessionId 的讨论页，When 路由解析，Then 不进入默认讨论页。

### 对应 FR-005 / FR-006
- Given 用户归档一个进行中会话，When 操作成功，Then 会话出现在已归档筛选中，不再出现在进行中筛选中。
- Given 用户重复归档同一个会话，When 请求再次提交，Then 返回当前已归档状态且不报错。
- Given 用户恢复可恢复会话，When 操作成功，Then 会话回到 `running` 状态。

### 对应 FR-007
- Given 用户在会话中心，When 点击新建入口，Then 跳转首页创建讨论入口。

### 对应 FR-008 / FR-009
- Given 首页存在最近讨论，When 用户查看首页，Then 最近讨论与会话中心数据一致。
- Given 用户点击首页最近讨论，When sessionId 有效，Then 进入对应 `/discussion/[sessionId]`。

### 对应 FR-011 / FR-012
- Given 用户进入讨论详情页，When 查看页面内容，Then 页面只展示当前会话消息流、角色栏和输入区，不展示会话列表、筛选或阶段评估入口。
- Given 用户查看底部导航，When 页面渲染完成，Then 只看到首页、会话、模板、设置四个一级入口。

### 对应 FR-013 / FR-014 / FR-015 / FR-016
- Given 新建会话处于 idle，When 讨论开始，Then phase 可进入 opening。
- Given 当前 phase 为 opening，When 开场完成并进入角色发言，Then phase 可进入 developing。
- Given 当前 phase 为 developing，When 满足基础推进规则，Then phase 可进入 climax。
- Given 当前 phase 为 climax，When 用户或系统要求收束，Then phase 可进入 closing。
- Given 任意 phase，When 请求非法跳转，Then 状态机拒绝跳转并保留原 phase。
- Given phase 进入 closing 且主持人总结消息已落库，When 用户人工确认完成，Then 会话标记为 `completed`。
- Given 发生合法 phase 或 status 变化，When 查询状态历史，Then 能看到对应变化记录。

### 对应 FR-017 / FR-018 / FR-019
- Given 调用会话列表查询能力，When 传入 status、keyword、limit，Then 返回匹配且按更新时间排序的会话。
- Given 调用状态更新能力，When sessionId 不存在，Then 返回 NOT_FOUND。
- Given 调用状态更新能力，When action 非法，Then 返回 VALIDATION_ERROR。
- Given 调用讨论状态查询能力，When sessionId 有效，Then 返回当前 phase 和状态历史。

## 5. Out of Scope

- 不实现用户意图分类、点名角色、快捷指令影响调度；这些属于迭代 5。
- 不实现 Director 主动邀请用户、分歧判断和自动收束策略；这些属于迭代 6。
- 不实现真实爽点事件触发、投票机制和事件卡业务逻辑；这些属于迭代 7。
- 不实现完整导出文件能力；导出入口可保留，但完整导出属于迭代 9。
- 不在讨论详情页新增独立阶段评估组件或跨会话管理能力。
- 不恢复底部“讨论”一级 Tab。
- 不新增模板、角色、Provider 或 Prompt 配置能力。

## 6. 依赖说明

- 依赖迭代 1 的会话创建能力：会话应具备 sessionId、topic、template 和基础状态。
- 依赖迭代 2 的多 Agent 运行时基础：讨论详情页继续展示角色与消息，不在本迭代重写 Agent 抽象。
- 依赖迭代 3 的讨论消息流联动：本迭代只扩展恢复、状态推进和生命周期，不重做消息 UI。
- 依赖统一会话数据源：会话中心、首页最近讨论和讨论详情页必须通过同一服务/仓储能力读取与更新会话。
